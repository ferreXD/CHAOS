# CHAOS Todo HTML View Contract

The human-digest view generated by `chaos:todo` is a **static HTML dashboard**, not Markdown.
Durable Markdown item files remain the only source of truth. The view is always regenerable from
`.chaos/todo/items/*.md` + `.chaos/todo/index.md` and must **never be hand-edited** — regenerate it.

## Generation is a deterministic script (not hand-authored)

The view is produced by a committed generator, so it can never drift from its source:

```bash
node tools/chaos-todo-views/generate.mjs
```

`chaos:todo` (any write that changes item files, and `chaos:todo --refresh`) regenerates the view
by **running this generator** — it does not hand-write HTML. Same inputs → byte-identical output.
See `tools/chaos-todo-views/README.md`. (Requires Node ≥ 22, already a CHAOS dependency; zero npm
deps.)

## Generated file

```text
.chaos/todo/views/index.html   # the whole dashboard: overview, by-target, by-type, blockers, all-items
```

One consolidated single-page dashboard with in-page navigation (tabs) replaces the former
per-target / per-type / blockers files. A dated snapshot may be produced on demand with
`--out .chaos/todo/todo-report-YYYY-MM-DD.html`. Only regenerate when source item files changed,
or when `--refresh` is passed (`policies.todo.htmlViews.regenerateOnlyWhenSourcesChange`).

## Hard requirements

- **Static & self-contained:** no server and no build step to view it — open the `.html` directly
  in a browser. **All** CSS and JS are inline in the document. **No external anything** — no CDN,
  no remote stylesheet/script/font/image, no network requests (`fetch`/XHR/WebSocket). It must work
  fully offline.
- **Inline JS is allowed** (this supersedes the earlier no-JavaScript rule) **only** when it is
  self-contained per the point above. The interactive layer (tab navigation, search, filter, sort,
  theme) renders from an embedded JSON blob of the parsed items. Reviewers review the **generator**
  (`tools/chaos-todo-views/`), not the emitted HTML.
- **Escaping is mandatory.** Every interpolated title, path, snippet, owner, and evidence excerpt
  is HTML-escaped (`&amp;`, `&lt;`, `&gt;`, `&quot;`, `&#39;`). The embedded JSON escapes `<`/`>`
  to prevent a `</script>` breakout. The client builds text via DOM/`textContent`; only a small,
  escape-first Markdown subset (`code`, `**bold**`, `*em*`, `-` lists) is ever turned into markup.
- **No secrets or sensitive content** — the same redaction discipline as every other CHAOS report.
- Include the generated-artifact comment immediately after `<!DOCTYPE html>`:

  ```html
  <!-- Generated by chaos:todo (tools/chaos-todo-views). Do not edit manually. Durable todo items live under .chaos/todo/items/. -->
  ```

## Required content (single-page dashboard)

The generated `index.html` presents, all reachable via in-page tabs from one file:

1. `<title>` and a generated-metadata line: generated timestamp, source command, and repository
   context (provider, branch, confidence) when resolved.
2. **Overview** — headline progress (done %), stat tiles (open, done, open blockers, stale,
   needs-decision), the **Recommended next** list (top 3–5), and the index provenance/context.
3. **By target** — grouped Public alpha / v1 / vNext / Later, each with its open·done split.
4. **By type** — grouped by item type.
5. **Blockers** — open `BLOCKER`-priority items (with an empty state when none).
6. **All items** — the full backlog with live search, status/priority/target filters, and sort;
   each item expands to Why / Next action / Closure criteria / Source evidence / History and links
   to its source Markdown file.
7. Footer: "Generated artifact — not source of truth. Source of truth: `.chaos/todo/items/`.
   Regenerate with `node tools/chaos-todo-views/generate.mjs`."

Stale items, duplicates, and coverage surface inside the relevant tabs (stale items carry their
missing-evidence reason; provenance/dedup notes come from `index.md`).

## Recommended-next ordering (deterministic)

Open items ranked by priority (`BLOCKER` > `HIGH` > `MEDIUM` > `LOW`), then nearest target
(`public-alpha` > `v1` > `vNext` > `later`), then `createdAt`, then `id`. Top 5. Implemented once,
in the generator.
