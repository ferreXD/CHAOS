# Gates Index — Task Tracker API

> Readiness checks. Gates evaluate meaningful readiness, not file presence. Each gate
> produces a labelled verdict (confidence + evidence_coverage + assumption_load).
> Generated by `chaos:init --guided` on 2026-07-19.

## G-01 — Proposal readiness (pre-implementation)

- **Purpose:** confirm a change is well-formed and its material decisions are answered
  before any code is written.
- **Required evidence:** an OpenSpec change (proposal + spec delta + tasks); every
  runtime decision for the change is `answered` (none `waiting`); rules checked.
- **Blocking criteria:** unresolved material decision; missing spec delta; contradicts an
  accepted decision without a new approved decision.
- **Deferrable:** non-critical open questions may be recorded and carried.
- **Owner:** human driver (via `chaos:review`).
- **Verdict shape:** `approved | rejected | changes-requested` + confidence metadata.

## G-02 — Apply scope integrity

- **Purpose:** ensure implementation stayed within the approved scope.
- **Required evidence:** `apply-report.md` with a scope-drift assessment; OpenSpec tasks
  checked off; code maps to approved tasks.
- **Blocking criteria:** scope expansion without a recorded decision; edits outside the
  change's declared surface.
- **Deferrable:** cosmetic follow-ups → todo backlog.
- **Owner:** `chaos:apply` + human.
- **Verdict shape:** `within-scope | drifted` + confidence metadata.

## G-03 — Verification (pre-archive)

- **Purpose:** independent confirmation that every requirement is implemented and tested.
- **Required evidence:** `verification.md` spec-traceability matrix (requirement → code →
  test); `dotnet test` green; `openspec validate` (standard+) with `--strict` where
  required.
- **Blocking criteria:** a behavioural requirement with no passing test; failing build/
  tests; broken traceability.
- **Deferrable:** `--light` may accept documented risk for non-critical gaps.
- **Owner:** `chaos:verify`.
- **Verdict shape:** `verified | not-verified` + confidence, evidence_coverage,
  assumption_load.

## G-04 — Archive & governance sync

- **Purpose:** close a change cleanly and keep durable governance coherent.
- **Required evidence:** G-03 passed; decisions audited; deltas promoted to base specs;
  debt/waivers recorded.
- **Blocking criteria:** archiving unverified work; unpromoted durable decisions; repo-
  wide sync attempted without maintainer/repo-owner confirmation.
- **Deferrable:** ADR promotion may follow via `chaos:sync` if recorded.
- **Owner:** `chaos:archive` + `chaos:sync`.
- **Verdict shape:** `archived | blocked` + confidence metadata.

## G-05 — Toolchain & runtime readiness (operational)

- **Purpose:** confirm the environment can actually run governed workflows.
- **Required evidence:** git, Node ≥ 20.19.0, npm, OpenSpec CLI present; `openspec/`
  project initialized; interaction runtime reachable (MCP) or explicit fallback.
- **Blocking criteria:** missing OpenSpec CLI or uninitialized `openspec/` blocks
  OpenSpec-dependent commands; malformed runtime state.
- **Deferrable:** optional tools (`.NET SDK` only needed to build/test the API) reported,
  not blocking for non-.NET commands.
- **Owner:** `chaos:doctor` / `chaos:status`.
- **Verdict shape:** `ready | needs-attention | blocked` + confidence metadata.
