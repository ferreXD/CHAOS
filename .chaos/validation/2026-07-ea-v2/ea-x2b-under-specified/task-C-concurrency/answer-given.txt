DECISION: Concurrency-control strategy for task updates.
ANSWER (maintainer): Selected option opt-optimistic — OPTIMISTIC concurrency, reject stale writes. Add a version token to tasks (starts at 1, increments on each successful update). A PUT that carries a stale/expected version that no longer matches the current version must be rejected with 409 Conflict and leave the task unchanged, so a client working from an out-of-date copy cannot silently clobber a newer edit; it must refetch and retry. Keep the existing version-less update path working for backward compatibility, and keep the build and tests green.
