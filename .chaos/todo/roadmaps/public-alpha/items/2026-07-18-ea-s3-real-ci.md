# TODO — EA-S3 — Real CI that runs the tests

## Why this exists

Tests exist but CI runs none of them, so a broken package cannot turn CI red. Contributor trust and drift protection depend on real CI.

## Source Evidence

- EA-S3 — Horizon 0, P0, complexity S. Depends on: EA-S2 (14-roadmap.md §14.2).

## Next Action

Wire CI to run tests and typecheck for all five packages, dotnet test for the C# example, and the parity check.

## Recommended Command

No specific CHAOS command required.

## Closure Criteria

- CI runs tests + typecheck for the 5 packages.
- CI runs dotnet test.
- CI runs the parity check.
- Any broken package turns CI red.

## Resolution

Added [.github/workflows/ci.yml](../../../../../.github/workflows/ci.yml):

- A `node` matrix job runs `npm ci` → `npm run typecheck` → `npm test` for all five packages (`tools/chaos-interaction-runtime`, `-mcp`, `-runner`, `-diagnostics`, `extensions/chaos-decision-center`). `fail-fast: false`, so every broken package is reported and any failure turns CI red.
- A `dotnet` job runs `dotnet test` for `examples/task-tracker/dotnet` (net8.0).
- Claude/Copilot surface parity stays enforced by the existing [.github/workflows/parity-check.yml](../../../../../.github/workflows/parity-check.yml).

Verified locally before commit: 266 Node tests (45 + 38 + 69 + 65 + 49) and typecheck green across all five packages, `dotnet test` green (5 tests), parity `RESULT: PARITY OK`. All four closure criteria met.
